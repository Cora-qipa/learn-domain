# 怎么跑评测 / Running Evals

> learn-domain 是交互式教学 skill,没有可断言的程序输出,所以"测试" = 触发准确性 + 流程质量两条线的人工/半自动走查。

## 两条测试线

### 线一:触发准确性(trigger-matrix.md)
验证 `description` 不漏触、不误触。

**手动**:把 `trigger-matrix.md` 里每条"输入"作为**新会话**首条消息,观察 learn-domain 是否被激活,对照期望。

**半自动(可选)**:用 `skill-creator` 的 eval 能力做 description 触发的方差分析——它能批量跑触发用例并给出稳定性报告。适合调 `description` 措辞后回归。

### 线二:流程质量(scenario-rubric.md)
验证走完五阶段不犯七个失败模式。

**手动走查**:扮演一个该领域的真实新手,完整走完一个"领域×模式",按 7 项量表打分。建议跑 `scenario-rubric.md` 里的三领域组合取均值。

## 各阶段交付前应跑的子集

| 阶段 | 跑什么 | 通过线 |
|---|---|---|
| P1 | 1 个领域完整走查 | 7 项里 ≥5 项满足 |
| P2 | 同领域回归,确认拆 references 后行为不退 | 与 P1 持平或更好 |
| P3 | 快变领域(如AI)+ 高风险领域(如医疗)各一次 | 第5、6项 + 高风险免责必须满足 |
| P4 | 触发矩阵全量 + 三领域质量均值 | 触发零误触;质量均值 ≥6/7 |

## 回归节奏
每次改动 SKILL.md 或 references 后,至少重跑一次"扩散模型/模式①"基线走查(见 examples/),确认没把已通过的行为改坏。

# 点线面体讲解引擎(阶段3)

> SKILL.md 阶段3 加载。这是 skill 的核心交付环节。按阶段1的依赖顺序,从地基概念起,逐个讲。每个概念走"点→线→(面/体)"的层次,用关卡制控制节奏。这一层是横纵分析法完全没有的——它是报告生成器,我们要的是教学引擎。

## 报告生成器 vs 教学引擎

| 横纵法默认(报告) | 本引擎(教学) |
|---|---|
| 一刀切,谁看都一样 | 从用户已懂的搭桥,因材施教 |
| 一次性倒完 | 讲一段→确认→再走(关卡制) |
| 默认用术语 | 显式"黑话→人话"翻译层 |
| 按教材/逻辑顺序 | 按依赖顺序 + 用户卡点排序 |
| 逐个概念讲清 | **概念对优先:先切边界,再各自展开(v2)** |
| 讲完就完 | 可选结业考验证真懂 |

## ⭐ v2 核心:讲解单元 = 概念对,不是单概念

**v1 的漏洞**:逐个概念讲清,"对比"是可选配菜。但初学者 80% 的困惑不在"A 是什么",而在"**A 和 B 到底差在哪**"。脑子里一团浆糊,需要有人拿刀切清边界。

**v2 的修正**:遇到阶段1 标注的**高混淆概念对**,讲解单元从"单概念"变成"概念对",且**边界优先**:

```
遇到高混淆对 (A, B):
  ① 先切边界(强制,前置!)
     "A 和 B 的本质区别:___ vs ___"
     一句话抓住那一刀,不是列5个维度让用户自己悟
     (详细话术见 concept-boundaries.md)

  ② 再各自展开
     A: 点 → 线-纵 →(线-横已在①切过,不重复)
     B: 点 → 线-纵

  ③ 关卡确认(验证边界真清楚)
     反问:"你能说出 A 和 B 最关键的不同吗?"
```

**关键变化**:
- "线-横"(对比)从可选配菜 → **必经环节,且前置**到各自展开之前。
- 对比重心从"参数表格" → "**一句话本质区别**"。
- 不等用户问,**主动**把最容易混的刀先切清楚。

> **遇到独立概念(无高混淆对)**:仍走下面的"单概念讲解层次",不必硬凑对子。

## 单个概念的讲解层次

### 点 — 概念内核(必给)
先给"啊哈"那一下,再挂细节:
- **一句话本质**:用最朴素的话说这概念到底是什么。不用术语解释术语。
- **一个好类比**:锚到用户已懂的东西(用诊断阶段找到的搭桥点)。类比是最强提速器。
  - 例:"RAG 之于大模型 = 开卷考试之于闭卷"。
  - **必须标边界**(见 anti-hallucination.md):"这是简化,它在 ___ 处不成立"。

### 线 — 横纵法(核心叙事层)
- **纵(它怎么来的)**:什么问题逼出了它。讲成 mini 故事(见 narrative-craft.md),不是年表。这一层补"为什么",让概念从黑话变活。
- **横(跟谁区分)**:跟用户最容易混的相邻概念对比,划清边界。这一层补"定位",防止概念搅成一锅粥。
  - **v2 强化**:横向对比不再可选。若该概念属于某个高混淆对,边界已在"概念对优先"流程的 ① 切过;若是讲到中途发现的新混淆点,立即切,用 `concept-boundaries.md` 的话术。
  - 重点对比"看起来像但其实不同"的:GAN vs 扩散、微调 vs RAG、股票 vs 债券。

### 面 / 体 — 按终点模式选配
- **面(全景定位)**:这概念在领域版图里的位置——模式①②需要。
- **体(怎么上手)**:最小动作、工具、新手坑——模式③需要。
不是每个概念都四维全上。模式①多数概念只需点+线。

### 分层挂载(防淹)
每个概念先给"够用版",然后挂一句可选深入:
> "这样够你看懂讨论了。想再深一层的话,它背后其实还有 ___,你要听我再展开。"
让用户自己决定钻不钻,绝不强行把所有"想深入"分支讲完。

## 关卡制(强制节奏,防一墙文字)

讲完一个概念,**停下来再走下一个**:
- **确认(每个概念后必做)**:一句轻量确认。可以是反问("你觉得它跟 X 的区别在哪?")或"这块清楚了我们继续?"。反问比"懂了吗"好——能暴露真实理解。
- **主动小测(可选)**:只在两种情况下出——① 用户主动要测;② 这是关键易错点、不测后面会塌。不要每段都测,会打断流畅感、显得啰嗦。

## 状态追踪(防重复、防跳跃)

全程记住:
- 已讲过哪些概念(讲过的不重讲,只在需要时引用回扣)。
- 用户在哪卡过(卡点优先回补)。
- 当前进度在依赖图的哪个位置。

如果用户的问题跳到了还没打地基的概念,温和地说"这个建立在 ___ 上,我们先把 ___ 过了再回来看它,会更顺",维持依赖顺序。

## 困惑驱动(进阶)

如果用户在某处明显卡住,可以暂时偏离预设顺序,先解决他当下的困惑——按"用户现在卡在哪"倒推该先讲什么,往往比死守教材顺序更快。解决后再回主线。

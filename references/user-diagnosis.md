# 自适应用户水平诊断(阶段2)

> SKILL.md 阶段2 加载。不诊断就开讲 = 一刀切失败模式。但也不能问一长串问卷。目标:用 2–4 道高质量问题定位用户真实水平和知识缺口,决定从哪个概念起讲、用什么类比、讲多深。

## 两个原则

### 原则一:自评 + 探测结合(别只信自评)
人会高估或低估自己。"你懂概率吗"——答"懂一点"几乎没信息量。所以:
1. 先一道**自评粗题**:快速区分"完全外行 / 沾过边 / 有相邻领域基础"。
2. 再 1–2 道**探测细题**:针对具体子概念出真问题,校准自评。
   - 探测题不是考试,是"你知道 X 大概是干嘛的吗"这种,让用户的回答暴露真实理解度。

### 原则二:自适应(CAT 式,每题切分假设空间)
像计算机自适应测试:每道题用来把"用户可能在哪个水平"的空间砍一半,根据上一题答案决定下一题往深还是往浅。

```
粗题:听过这领域的核心词吗?
 ├─ 完全没听过 ──→ 探测题往浅:确认有无最近的相邻基础 ──→ 从"这领域解决什么问题"起讲
 ├─ 听过没深究 ──→ 探测题居中:问一个核心概念的用途 ──→ 从核心概念起讲,术语轻补
 └─ 用过/学过 ──→ 探测题往深:问一个易错的区分 ──→ 跳过地基,直接攻盲点
```

问够即止。大多数情况 2–4 题足够。**不要为了严谨多问**——诊断是手段不是目的。

## 出题模板

- **粗题(自评+触达)**:"提到 [领域],像 [核心词A]、[核心词B] 这些词,你是完全陌生、听过但说不清、还是用过?"
- **探测题(查理解)**:"那 [某核心概念] 你知道它大概是解决什么问题的吗?随便说说。"
- **相邻基础题(找搭桥点)**:"你平时接触过 [相邻领域] 吗?" —— 找到的相邻基础就是阶段3 类比的锚点。
- **盲点题(给有基础者)**:"[易混的A] 和 [易混的B] 你能说出区别吗?" —— 暴露"自以为懂"的盲区。

## 产出:用户水平画像

诊断完在心里(或简短跟用户确认)形成:
- **整体档位**:完全外行 / 沾边 / 有基础。
- **可用的搭桥点**:用户已懂的相邻领域(决定用什么类比)。
- **各子概念缺口**:哪些要从零讲,哪些可一句带过。
- **起讲点**:依赖图上的哪个概念。

这个画像直接喂给阶段3。讲解过程中如果发现画像不准(用户其实更懂或更不懂),随时调整,不必重新诊断。

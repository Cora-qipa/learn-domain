# 测绘领域地形(阶段1)

> SKILL.md 阶段1 加载。横纵分析法直接跑在"一个对象"上(Cursor、Anthropic),但领域是一片场,里面有很多对象、概念、流派。不能像分析一个产品那样分析"分布式系统"。所以必须先把场拆开测绘——这是横纵法缺的前置步,也是"先地图后细节"信条的落地。

## 目标

给用户一张**精简地图**,让他在听任何细节前先有全局,新知识才有地方挂。地图不是讲解,是骨架——这一步不展开讲任何概念,只画结构。

> **⭐ 地图是本产品的核心交付物,不是过场。** 本 skill 的价值主张是"系统化的框架认知",而那个"框架"具象出来就是这张地图。地图的质量 = 产品的质量:一张漏了主干、关系混乱的地图,后面讲得再好也救不回来。所以这一步要舍得花力气——宁可地图本身打磨到位,也不要急着往下讲。讲解(阶段3)是在给这张地图"填色",地图本身才是用户带走的那张认知底图。

## 四个动作(v2 新增:标注边界网)

### 1. 拆解:把领域切成关键概念 / 玩家 / 流派
- **概念**:这领域绕不开的核心名词(如机器学习的"监督/无监督/损失函数/过拟合")。
- **玩家**:领域里的主要产品/公司/工具/人物(如 AI 领域的 OpenAI/Anthropic/Google)。
- **流派**:思想阵营或方法路线(如 AI 里的"符号主义 vs 连接主义",投资里的"价值 vs 成长")。
按学习终点取舍:模式①重概念+流派,模式②重玩家+对比维度,模式③重概念+工具。

**v2 新增**:拆解时不只列概念,还要识别**概念对**(哪些概念会一起出现、需要对比的)。

### 2. 排依赖图:谁必须在谁之前学
这是入门最值钱的产出。新手最大的痛是不知道**先学什么**。画一条依赖链:

```
基础直觉A ──→ 核心概念B ──→ 应用C
              └──→ 核心概念D ──→ 应用E
```

原则:
- 把"理解 X 必须先懂 Y"的 Y 排在前面。
- 找到**地基概念**(没有前置、且很多东西依赖它)——从这里起讲。
- 依赖图也暴露出哪些是"叶子"(可以最后讲或跳过)。

### 3. 标 80/20 最小集
在依赖图上圈出那 20% 解锁 80% 领域的概念。判断标准:
- 被其他概念依赖得最多的(枢纽节点)。
- 在该领域的文章/讨论里出现频率最高的。
- 不懂它就完全读不下去的"门槛词"。
其余概念先挂起(grey out),用户问到或结业考暴露缺口再补。

### 4. 标注概念边界网(v2 核心新增)

**目标**:识别概念对的关系 + 混淆风险,在地图上显式画出"边界"。

**操作步骤**:
1. **识别关键概念对** — 找出那些"会一起出现、新手需要对比"的概念组合
   - 并列候选:功能相似、经常被拿来比较的
   - 包含候选:名字里有"大小"关系暗示的
   - 层级候选:有"控制/依赖"关系的
   - 易混候选:名字相似、口语混用的

2. **标注关系类型**(每对标一个,详见 `concept-boundaries.md`)
   - ① 并列:同级,各干各的(如 CPU ∥ GPU)
   - ② 包含:谁装谁(如 GPU ⊂ 显卡)
   - ③ 层级:谁依赖谁(如 CUDA → GPU)
   - ④ 易混淆:关系不明显但常混(兜底)

3. **判断混淆风险**(高/中/低,用启发式)
   - **高**:名字相似 / 功能重叠 / 口语混用 / 包含关系但常被误当并列
     → 阶段3 **强制切、优先切**
   - **中**:关系不够直观
     → 阶段3 轻提
   - **低**:关系明显
     → 可选,用户问才展开

**示例标注**(某领域片段):
```
概念对列表:
  (CPU, GPU)      → ① 并列, 混淆风险:高 ⚠️
  (GPU, 显卡)     → ② 包含, 混淆风险:高 ⚠️
  (显存, 内存)    → ① 并列, 混淆风险:中
  (CUDA, GPU)    → ③ 层级, 混淆风险:中
```

这张"概念对+关系+风险"清单,就是**边界网的数据结构**,下一步可视化会用到。

## 给用户看什么(v2 增强:依赖图 + 边界网)

### 基础版:依赖链(保留 v1)
一张**轻量地图**,文字版依赖链或简单 ASCII:

```
[领域] 学习路径
  ① 地基:____(先打这个)
  ② 核心:____ → ____(80/20 就在这)
  ③ 进阶:____(想深入再说)
  主要流派/玩家:A 派(主张…) vs B 派(主张…)
```

### v2 增强:边界网(可选,复杂领域推荐)
在依赖链基础上,加一张**概念关系图**,显式画出边界:

```
[领域] 概念关系图
  
  CPU ═══(并列)═══ GPU      ⚠️ 高混淆:少而精 vs 多而傻
   │                │
  (层级)          (包含)
   ↓                ↓
  内存            显卡 ⊃ [GPU + 显存]  ⚠️ 高混淆:芯 vs 板
                    │
                  (层级)
                    ↓
                  CUDA(软件) → GPU(硬件)
```

**关系符号约定**:
- `═══` : 并列
- `⊃` / `⊂` : 包含
- `→` : 层级/依赖
- `⟷` : 易混淆
- `⚠️` : 高混淆风险,阶段3 强制切

**何时用边界网**:
- 概念多(>6 个)且关系复杂 → 推荐边界网
- 概念少(<5 个)且关系直观 → 依赖链即可
- 高混淆对 ≥2 个 → 推荐边界网

配一句话:"我们会沿这条线走,先打地基 ①,再攻核心 ②。**图里标 ⚠️ 的是最容易混的概念对,我会优先帮你切清楚。**"

## 诚实边界声明(给完地图后【必做】)⭐

这是本产品的**承诺底线**:我们卖"系统化框架",但方法只能保证"组织得好"(结构/顺序/区分),**保证不了"内容没漏"**。模型即兴生成的地图,一样可能漏掉某个重要流派或分支。一张**看起来很系统、实则有洞**的地图比零散提问更危险——用户会误以为自己已经全面了,反而不去补。

所以给完地图后,**必须**补一句声明,不能跳过。模板:

> "这是这个领域的**入门主干**框架,帮你先建立结构和顺序。我不敢保证覆盖了所有分支——
> ① 你要是有特别想了解、或听说过但图上没有的方向,告诉我,我补进来;
> ②(若环境能联网)我也可以对照该领域的综述/教材大纲帮你查漏补缺。"

这句话同时做三件事:**不假装全面**(诚实,信任不崩)、**留口子让用户补**(人机共建,反补盲区)、**可选联网校准**(能联网时用真实综述兜底)。

**适配不同领域**:
- 慢变领域(数学/经典理论):重点在"① 你想深入哪个分支"——内容稳定,主要靠用户指方向。
- 快变领域(AI/政策/市场):重点在"② 联网查漏"——内容变得快,模型记忆可能过期,优先联网校准。

**别啰嗦**:这句声明只在阶段1 给完地图后出现**一次**,不要每个概念都念。

### 边界网绘制指南(原子模板)

不会一次画对整张大图。先用下面四个**原子模板**拼出每对关系,再组合:

**① 并列**(横向并排,等号连接):
```
A ═══ B        或带一刀本质:  A ═══ B
                               少而精  多而傻
```

**② 包含**(用 ⊃ 或嵌套方括号):
```
B ⊃ [ A + 其他 ]      例:显卡 ⊃ [ GPU + 显存 + 供电 ]
```

**③ 层级**(竖向箭头,上指挥/依赖下):
```
A              例:  CUDA(软件)
↓ (指挥/依赖)         ↓
B                    GPU(硬件)
```

**④ 易混淆**(双向箭头 + ⚠️):
```
A ⟷ B  ⚠️      例:  训练 ⟷ 推理  ⚠️ 学 vs 用
```

**组合成网的三条纪律(防止图失控)**:
1. **一张图最多 7 个概念**。超了就拆成两张,或只画 80/20 最小集那几个。
2. **高混淆对必须上图并标 ⚠️**;低混淆对可以省略,别堆满。
3. **每条边尽量带"一刀本质"**(那句区别),光画线不写区别等于没切。

**降级规则**:如果 ASCII 图画出来超过 12 行、或自己都觉得乱,**退回纯文字列表**(就是上面"示例标注"那种 `(A,B)→关系,风险` 清单)。清楚 > 好看。

## 可选:用真实文章校准地图

如果用户开场自带了一篇看不懂的文章,可以用它**反推该教什么**——提取文章里出现的概念,确保地图覆盖了它们,让讲解不悬空。但这是可选增强,不是必须;主线仍是按领域本身的依赖结构测绘。

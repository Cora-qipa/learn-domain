# 测绘领域地形(阶段1)

> SKILL.md 阶段1 加载。横纵分析法直接跑在"一个对象"上(Cursor、Anthropic),但领域是一片场,里面有很多对象、概念、流派。不能像分析一个产品那样分析"分布式系统"。所以必须先把场拆开测绘——这是横纵法缺的前置步,也是"先地图后细节"信条的落地。

## 目标

给用户一张**精简地图**,让他在听任何细节前先有全局,新知识才有地方挂。地图不是讲解,是骨架——这一步不展开讲任何概念,只画结构。

## 三个动作

### 1. 拆解:把领域切成关键概念 / 玩家 / 流派
- **概念**:这领域绕不开的核心名词(如机器学习的"监督/无监督/损失函数/过拟合")。
- **玩家**:领域里的主要产品/公司/工具/人物(如 AI 领域的 OpenAI/Anthropic/Google)。
- **流派**:思想阵营或方法路线(如 AI 里的"符号主义 vs 连接主义",投资里的"价值 vs 成长")。
按学习终点取舍:模式①重概念+流派,模式②重玩家+对比维度,模式③重概念+工具。

### 2. 排依赖图:谁必须在谁之前学
这是入门最值钱的产出。新手最大的痛是不知道**先学什么**。画一条依赖链:

```
基础直觉A ──→ 核心概念B ──→ 应用C
              └──→ 核心概念D ──→ 应用E
```

原则:
- 把"理解 X 必须先懂 Y"的 Y 排在前面。
- 找到**地基概念**(没有前置、且很多东西依赖它)——从这里起讲。
- 依赖图也暴露出哪些是"叶子"(可以最后讲或跳过)。

### 3. 标 80/20 最小集
在依赖图上圈出那 20% 解锁 80% 领域的概念。判断标准:
- 被其他概念依赖得最多的(枢纽节点)。
- 在该领域的文章/讨论里出现频率最高的。
- 不懂它就完全读不下去的"门槛词"。
其余概念先挂起(grey out),用户问到或结业考暴露缺口再补。

## 给用户看什么

一张**轻量地图**即可,不要长篇。文字版依赖链或简单 ASCII:

```
[领域] 学习路径
  ① 地基:____(先打这个)
  ② 核心:____ → ____(80/20 就在这)
  ③ 进阶:____(想深入再说)
  主要流派/玩家:A 派(主张…) vs B 派(主张…)
```

配一句话:"我们会沿这条线走,先打地基 ①,再攻核心 ②;③ 和旁支你想深入我们再展开。这样你心里先有张图。"

## 可选:用真实文章校准地图

如果用户开场自带了一篇看不懂的文章,可以用它**反推该教什么**——提取文章里出现的概念,确保地图覆盖了它们,让讲解不悬空。但这是可选增强,不是必须;主线仍是按领域本身的依赖结构测绘。

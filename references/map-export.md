# 地图导出 / 留存(阶段5)

> SKILL.md 阶段5 加载。本 skill 的核心价值是"系统化的框架认知",而那个框架具象出来就是地图——所以要让用户能**把地图带走**,而不是关掉对话就没了。这一步把地图沉淀成一份结构化、可保存、可回看、可分享的"领域地图卡片"。

## 统一模板:领域地图卡片

无论哪个运行环境,导出物都用这一个模板,保证一致。整理阶段1 的地图 + 全程沉淀的术语,填进下面结构:

```markdown
# 📍 领域地图 · <领域名>

> 学习终点:<① 看懂/会聊 | ② 会判断 | ③ 会上手>
> 生成:learn-domain · <日期>

## 一、学习路径(依赖顺序)
① 地基:____(先打这个)
② 核心:____ → ____(80/20 最小集就在这)
③ 进阶:____(想深入再补)

## 二、概念边界网(谁和谁怎么区分)
<把阶段1 那张边界网放这里。每条边带"一刀本质区别">
A ═══ B    ⚠️ 本质区别:___ vs ___
C ⊃ [D+E]  本质区别:___
F ──→ G    本质区别:___

## 三、术语表(黑话 → 人话)
| 术语 | 一句话人话 |
|---|---|
| ____ | ____ |

## 四、边界声明(诚实)
这是入门**主干**框架,可能未覆盖的分支:____。
想继续深入可往这些方向补:____。

## 五、下一步
- 推荐先读:____
- 想验证掌握:找一篇该领域真实文章试读
```

## 怎么导出(按运行环境分流)

### A. 能写文件的环境(Claude Code 等)
1. 阶段5 主动问:"要不要我把这张领域地图存成文件,方便你以后回看?"
2. 用户同意 → 用上面模板生成完整卡片,**写入文件**:
   - 默认路径:`./learn-domain-<领域名>.md`(当前目录)
   - 或用户指定的路径
3. 写完告诉用户文件位置,一句话即可。
4. 不要未经同意就写文件——先问再写。

### B. 不能写文件的环境(通用 prompt 版 / 网页模型)
1. 阶段5 主动说:"我把这趟的领域地图整理成一张卡片,你可以复制保存下来。"
2. 用上面模板生成完整卡片,**放进一个 ```markdown 代码块**里输出——方便用户一键复制。
3. 提示用户:"复制这段存到你的笔记里(Notion/备忘录/Obsidian 都行),下次想回顾或继续深入时,把它再贴给我,我就能接着上次的框架带你往下走。"

## 关键原则

- **卡片要自包含**:用户单看这张卡片,不依赖对话上下文也能回忆起整个框架。
- **边界网是卡片的灵魂**:术语表别的工具也有,但"概念之间怎么区分"这张关系网是本产品独有的核心资产,务必完整。
- **诚实声明必须在**:第四节"边界声明"不能省——这是本版定位(系统化≠全面)的兑现。
- **别过度**:卡片是浓缩的骨架,不是把讲解全文塞进去。控制在一屏到两屏内。

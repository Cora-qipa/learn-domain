# 学习终点模式详解

> SKILL.md 阶段0 加载。学习终点是整个 skill 的总开关——输入从"文章"改成"领域"后失去了天然范围边界,用"用户学了想干嘛"重新锚定边界:它决定讲多深、讲哪些、怎么考、点线面体哪一维加重。

## 为什么必须最先锁终点

领域是无底洞。"入门机器学习"——给想看懂新闻的人和想转行的人,需要的 20% 完全不同。不锁终点,讲解必然失焦、讲不完。终点同时顶替了被砍掉的文章原本扛的两根支柱之一(范围边界;另一根"防幻觉锚"由 anti-hallucination.md 顶替)。

## 三种模式

### ① 看懂 / 会聊型
- **用户想干嘛**:读懂这领域的新闻/文章/帖子,跟圈内人聊天不露怯、能问对问题。
- **深度**:能认能懂、知道各方在吵什么。**不要求会用、会做**。
- **最小集偏重**:高频术语 + 核心心智模型 + 流派版图(谁和谁是两个阵营)。
- **点线面体重心**:点(内核+类比)+ 线(横纵),面只给轻量版图。
- **结业考**:读懂一篇该领域近期优质文章。
- **合并说明**:原"看懂型"和"会聊型"合并——读懂内容和听懂讨论需要的底座高度重合,拆开是过度设计。

### ② 会判断型
- **用户想干嘛**:要做个决定——技术选型 / 评估方案 / 要不要投资或采购。
- **深度**:各方案的 tradeoff + 评判标准,够支撑一个有理有据的决定。
- **最小集偏重**:关键决策维度 + 选项横向对比。
- **点线面体重心**:线(对比是核心)+ 面(全景定位)。
- **结业考**:给一个真实场景,让用户做判断并说出理由。

### ③ 会上手型
- **用户想干嘛**:真动手——写第一行代码 / 跑通第一个流程 / 为转行打基础。
- **深度**:到可操作,含工具、环境、最小实践。
- **最小集偏重**:地基概念 + 最小可行实践路径(能跑起来的 hello-world)。
- **点线面体重心**:体(上手)+ 面(路径)。
- **结业考**:完成一个最小实操。

## 引导选择话术(用户没想清楚时)

不要甩一张表让用户研究。问一句口语化的:

> "你学 [领域] 主要想干嘛——是想**读懂**相关的内容和讨论,还是想**做个判断/决定**,或者想**真正上手**做点东西?"

根据回答归类。模糊回答的处理:
- "就想了解一下" → 默认①,但追一句"是想能看懂别人讨论,还是以后自己要用?"
- "工作要用" → 区分是②(做决定)还是③(自己执行)。
- 用户跨多个 → 选最主要的一个作为本次主线,告诉他"我们先打通①的底座,这也是②③的地基,后面你想往哪深挖都行"。

## 锁定后必做:复述边界

确认终点后,用一句话跟用户复述,这句话是后面所有取舍的依据:

> "那我们的目标是让你能 ___,讲到 ___ 为止,够用就停;想深入的地方我会标出来,你说要才展开。"

后续任何"要不要讲这个"的犹豫,都回到这句边界裁决。

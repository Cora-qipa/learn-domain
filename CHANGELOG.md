# Changelog

All notable changes to this skill are documented here.

## [2.2.0] — 2026-06-02

**定位校正 / Positioning Fix**。把核心承诺从兜不住的「全面」精确为站得住的「系统化框架」。这一版以减法和校正为主,不加重交付能力(点线面体/横纵法/边界机制/五阶段全部保留)。

> **为什么改**:产品反复在"快速入门"和"深度理解"间拧巴,根因是承诺口径错位。方法(点线面+横纵)能保证输出"组织得好"(结构/顺序/区分),但**保证不了"内容没漏"**——"全面"靠模型即兴生成,迟早被用户发现漏块,信任崩塌。改成承诺"系统化框架",承诺就兑现得了,且卖点几乎不损(用户痛的就是"提问太零散没系统")。

### Changed
- **核心定位**:承诺从「全面认知」→「**系统化的框架认知**」(一张有结构、有顺序、有概念边界的领域地图)。统一到 `SKILL.md` / `README.md` / `dist/learn-domain-prompt.md` / `dist/README.md` 四处口径。
- **价值主张重写**:点明"用户自己即兴提问只能盲人摸象,本 skill 用固定方法逼出一个系统化的下限"——护城河是方法保证的下限,不取决于用户会不会问。
- **地图升为第一交付物**:从阶段1 的"过场"提升为产品核心资产。地图质量 = 产品质量;阶段5 沉淀清单里地图排首位。

### Added
- **诚实边界声明(强制)**:阶段1 给完地图后,必须主动声明"这是入门**主干**框架,可能没覆盖所有分支"——① 留口子让用户补充想了解的方向;②(能联网时)对照领域综述查漏。堵住"系统化的残缺品"这个最大风险。
- **地图留存/导出** `references/map-export.md`:统一的"领域地图卡片"模板。能写文件的环境(Claude Code)经用户同意写入 `./learn-domain-<领域名>.md`;不能写文件的环境(通用 prompt 版)输出一张可复制保存的卡片。两版字段一致。

### Notes
- 明确不做(防 scope creep):强制联网校准框架(只做可选查漏)、动横纵法/边界/五阶段引擎。
- v3 计划:跨会话记忆 / 插件市场可订阅 / 领域混淆风险库。

## [2.1.0] — 2026-06-02

**跨模型可用 / Universal Prompt**。让 learn-domain 脱离 Claude 生态,任何模型都能用。同时 v2 核心机制稳定,去掉 -beta 转正。

### Added
- **通用 Prompt 单文件** `dist/learn-domain-prompt.md`:把 SKILL.md + 8 个 reference 揉成一份自包含、连贯的 prompt。复制粘贴进 GPT / Gemini / 豆包 / Kimi / DeepSeek / 文心 等任意模型即可使用,无需 Claude。
- **dist 使用说明** `dist/README.md`:给非 Claude 用户的三步上手 + 各模型注意事项(联网/离线)。
- 主 README 新增"在其他模型上使用"章节(中英双语)。

### Changed
- 通用版的联网核查改为**优雅降级**:能联网的模型自动查证,不能联网的明确声明"建议自查"而非编造。
- 版本号 `2.0.0-beta` → `2.1.0`(v2 概念边界机制经设计验证稳定,正式转正)。

### Notes
- v2.1 后续:真实用户(跨模型)走查反馈 → 沉淀领域混淆风险库。
- v3 计划:跨会话记忆 / 插件市场可订阅。

## [2.0.0-beta] — 2026-06-02

**概念边界优先 / Concept Boundary-First**。基于 v1 真实用户反馈("不够本质,不会主动理清 CPU 和 GPU 的区别")。

### Added
- **概念边界机制**:新增 `references/concept-boundaries.md`——四类概念关系(并列/包含/层级/易混淆兜底)+ 混淆风险判断(高/中/低启发式)+ 边界切分话术库。
- **概念边界网**:阶段1 测绘时显式标注概念对的关系与混淆风险,并用 ASCII 图可视化(四个原子模板 + 防失控三纪律)。

### Changed
- **讲解单元从"单概念"变成"概念对优先"**:遇到高混淆对,强制**前置切边界**(一句话本质区别)→ 再各自展开 → 关卡确认。
- "线-横"(概念对比)从可选配菜**升为必经环节**。
- 失败模式新增一条:"高混淆概念对不主动切边界,等用户问才对比"。

### Notes
- v2.0 正式版:待真实用户走查通过后去掉 -beta。
- v2.1 计划:沉淀领域混淆风险库。
- v3 计划:跨会话记忆 / 插件市场可订阅。

## [1.0.0] — 2026-06-01

首个正式发布版本。First public release.

### Added

- **五阶段工作流**:定终点 → 测绘地形 → 诊断水平 → 点线面体讲解 → 可选结业考。

- **测试套件**:触发矩阵、流程评分量表、评测运行说明。
